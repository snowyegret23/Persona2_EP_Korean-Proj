# Persona 2: Eternal Punishment 한국어 번역 프로젝트

[![Video Label](http://img.youtube.com/vi/cTqGIsIDrPA/0.jpg)](https://youtu.be/cTqGIsIDrPA)

본 저장소는 **PSP판 Persona 2: Eternal Punishment(일본판)**의 한국어 번역 작업을 위한 데이터/스크립트와 빌드 도구를 제공합니다.

---

## ⚠️ 저작권 및 권리 불행사 고지 (필독)

- Persona 2: Eternal Punishment 및 관련 모든 원저작물의 권리는 원권리자(Atlus 및 관련 권리자)에게 있습니다.
- 본 프로젝트 팀/기여자는 원저작물에 대한 소유권을 주장하지 않습니다.
- 본 저장소는 비영리 팬 번역 및 기술 연구 목적의 자료만을 다룹니다.
- **본 프로젝트는 원저작물, 캐릭터, 상표, 사운드, 이미지, 프로그램 코드 등 어떤 저작권/상표권/지식재산권도 행사하지 않으며, 행사할 의사도 없습니다.**
- 게임 원본 파일(ROM/ISO/EBOOT 등)은 각 사용자가 합법적으로 직접 구해야 하며, 저장소/기여자는 이를 배포하지 않습니다.

---

## 프로젝트 구조(요약)

- `p2_kor/`: 한국어 모드 데이터(텍스트, 심볼, 설정)
- `p2_tool/`: 빌드/패치 도구(TypeScript)
- `p2_text/`: 원문/패치 관련 텍스트 리소스

## 번역 시작하기

### 1) 준비물

- Windows 환경(배치 파일 실행 기준)
- Node.js / npm
- 프로젝트 클론
- **사용자가 직접 구한 일본판 원본 ISO**

### 2) 원본 ISO 배치 (중요)

`p2_tool/Build.bat` 기준으로, 원본 ISO는 아래 규칙을 반드시 따라야 합니다.

- 위치: **저장소 루트** (`p2_tool` 폴더의 상위 경로)
- 파일명: **`Persona 2 - Batsu - Eternal Punishment (Japan).iso`**

즉, 최종적으로 아래처럼 배치되어야 합니다.

- `./Persona 2 - Batsu - Eternal Punishment (Japan).iso`

> 파일명이 다르거나 경로가 다르면 `Build.bat`가 그대로는 동작하지 않습니다.

### 3) 의존성 설치

```bash
cd p2_tool
npm install
```

### 4) 빌드 실행

```bat
cd p2_tool
Build.bat
```

빌드 결과물은 기본적으로 `p2_build` 경로 아래 생성됩니다.

## 테스트 진행 방법

번역은 텍스트 변경이라도 실제 게임 내 문맥/줄바꿈/제어코드 영향이 있으므로, 아래 순서대로 확인을 권장합니다.

### A. 빌드 성공 여부 확인

- `Build.bat` 실행 시 에러 없이 완료되는지 확인
- `p2_build` 산출물 생성 여부 확인

### B. 기본 동작 테스트(에뮬레이터)

1. 빌드 결과물을 PPSSPP 등에서 실행
2. 수정한 이벤트/대사 진입 지점까지 진행
3. 다음 항목 점검
   - 텍스트 깨짐 여부
   - 줄바꿈/말풍선 넘침 여부
   - 제어코드로 인한 UI 비정상 동작 여부
   - 진행 불가(소프트락) 여부

### C. 회귀 테스트(최소)

- 수정 구간 전후의 대화/이벤트도 함께 점검
- 전투 진입/메뉴 진입/저장-불러오기 후 동일 현상 재확인

## 크레딧 및 감사의 글

- 본 프로젝트의 기반이 된 툴은 **casserowl(eiowlta)**님이 처음 제작하셨습니다.
- casserowl님은 이 게임의 영어 패치를 제작하신 분으로, 단순히 기존 영어 패치용 툴을 공유한 것에 그치지 않고 본 프로젝트에도 큰 도움을 주셨습니다.
